# 카드뉴스 자동화 본질 1쪽 (anchor)

이 문서가 모든 prompt·evaluator·cron·critic 룰의 anchor다. 의심 생기면 여기 먼저 보고 코드 본다.

> 출처: 메모리 4건(`feedback_essence_before_engineering`, `feedback_automation_value_definition`, `feedback_cardnews_essence_first`, `project_automation_essence_dimension_b`) + `clients.brand_voice` + 비전 보고서(2026-04-28) + 차원 B 진단(2026-04-30) 통합. 갱신 trigger: 사용자 또는 7일 KPI 누적 결과.

## 1. 가치 정의 — LM 1장이 무엇을 만드는가 (client별 1문장)

### fit_ai_founder
> **23살 예비 사업가가 AI 도구를 *직접 1주일 써본 결과 수치*를 1슬라이드 1메시지로 보여줘서, 같은 처지의 20대 예비창업가·소상공인이 자기 도구 결정을 빠르게 내리게 한다.**

남이 못 만드는 것 = **본인의 1차 실증 결과**. 이게 빠지면 도구 비교 일반론 = AI 슬롭 → 가치 0.

### planb_pm
> **분당·판교 단일 매물·단지의 호가 vs 실거래 vs 평형별 격차를 1슬라이드 1수치 + 출처로 보여줘서, 상위 5% 자산가가 본인 단지 의사결정을 데이터로 검증하게 한다.**

남이 못 만드는 것 = **단지·매물·실거래 1차 데이터 + 권위 에디토리얼 톤**. 일반 부동산 뉴스 요약 = 가치 0.

## 2. 측정 3종 (보조 지표 노이즈 흡수 후 진짜 KPI)

| 지표 | 의미 | 1차 가설 임계 (7일 누적 후 갱신) |
|---|---|---|
| **저장률** = 저장수/도달 | "다시 볼 가치 있는가" — 진짜 KPI | ≥ 5% (10명 보면 1명 저장) |
| **DM 전환률** = (DM 또는 키워드 코멘트)/도달 | 행동 전환 | ≥ 0.5% (1000명 도달 시 5명) |
| 도달 = 노출/팔로워 | 알고리즘 체급. 보조 | ≥ 본인 팔로워 × 0.3 |

vision_score / critic_total / design_quality_score 는 **선행 지표**일 뿐 KPI 아님. ±5점 노이즈 가능. 7일 누적 위 3종이 진짜.

미달 시 1차 진단:
- 저장률 ↓ → 1차 데이터·수치·출처 결여 (양산만 한 흔적)
- DM 전환 ↓ → CTA·hook·키워드 결함
- 도달 ↓ → hook 첫 1초 stop 실패

## 3. 양산 vs 인간 — 차원 B 운영 룰

| 단계 | 코드 양산 | 인간 필수 |
|---|---|---|
| 1차 데이터 수집 | ❌ **금지** (AI 추측 = 환각 = 슬롭) | ✅ **사용자가 1차 결과 직접 입력** |
| 토픽 후보 5종 (5신호) | ✅ 자동 | ✅ 1개 선택 (5초/일) |
| 콘텐츠·디자인·캡션 생성 | ✅ 자동 (단, 1차 데이터에 종속) | ❌ |
| 게시 승인 | ❌ | ✅ design_ready 슬랙 1클릭 |
| 분석·brand_voice 학습 | ✅ 자동 | ✅ 승인/거부 신호 |

핵심: **1차 데이터 슬롯이 비어있으면 콘텐츠 생성 자체를 SKIP**. AI가 "그럴듯한 일반론"으로 빈 칸 채우는 순간 슬롭 발생.

## 6. 자기비판 (이 문서의 한계)

- 임계 5%/0.5%는 1차 가설. 실데이터 누적 후 5/95 percentile로 갱신 필요.
- "1차 데이터 입력 슬롯"이 사용자 부담 ↑ 트레이드오프 — 일주일 5분 입력으로 줄여야 운영 지속.
- planb_pm은 사용자 본인 1차 매물 정보 외에도 KB·네이버 외부 소스 자동화 가능 (트레이드오프 별도 검토).
- vision_score는 *측정 신뢰도 낮음 인정* — 단일 게이트 차단으로 안 쓰고, 콘텐츠 생성 단계 self-check에만 사용.
